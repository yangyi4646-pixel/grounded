# 信息归属 · Canonical 定义(单一权威源)

> 这是「信息归属 / 产品结构」5 项的**唯一权威定义**。prd-to-spec 与 spec-to-code 两处都引用本文件,不再各自复述细节——改定义只改这里,两处随之更新,杜绝两处描述漂移。
> 触发说明、机械检查脚本、产出模板各自留在原处(见文末「触发与机械检查」);本文件只定义「每一项是什么」。

## 主对象(Current page main object)
当前页面/视角真正承载的那个对象或主关系。整页信息层级都围绕它组织:谁是「行/记录」的载体、头部摘要在摘要谁、展开区在补充谁、详情页在承载谁,全部以主对象为锚。判主对象错 = 整页归属错位的根因,所以这是 load-bearing 项,**不允许空、不允许 `N/A`**。

## 信息归属矩阵(Information Ownership Matrix)
把页面上每一条信息按「归属层」分类的网格,逼出「谁拥有这条信息、它该在哪一层展示」。矩阵必须回答这 6 个问题:
- 当前页面**主对象**是什么?
- 哪些信息属于**当前主对象**?
- 哪些信息属于**上游对象**(在当前对象之前或之外独立存在)?
- 哪些信息属于**下游消费对象**(只有被其它 surface 消费时才成立)?
- 哪些信息只在**两个对象发生关系**时才成立(关系态信息)?
- 哪些信息只是**完整详情页**的信息,不应在当前视角重复展示?

每行必须有具体信息、展示职责、去重/排除规则和来源;空格 = 未决 gap。

## 头部摘要职责(Header summary responsibility)
头部摘要这一层**负责展示什么**的边界定义:它承载主对象的概览级字段、映射或同义语义(让用户一眼认出「这是谁、当前态如何」)。一旦某字段/映射在头部出现,它就**归头部所有**,下游各层不得再重复同一信息。

## 展开区职责(Expanded-detail responsibility)
展开/收起区这一层**负责展示什么**的边界定义:展开区只回答「头部没有给出的**新增**信息是什么」。它不是头部的扩写或重排,不复述头部已展示的字段、映射或同义语义。展开区无新增信息时,不应为展开而展开。

## 不重复展示规则(Do-not-repeat display rule)
跨层去重的硬规则,防同一信息在多层冗余:
- 一个信息只能出现在**最合适的一层**。
- 头部摘要已展示的字段、映射或同义信息,展开区**不得**用同名或换一种说法重复。
- 展开区只回答:头部没有给出的新增信息是什么。
- 完整详情页信息通过**详情页链接/跳转**承载,不在当前视角复刻。

## 触发与机械检查(引用,不在此复述)

- **prd-to-spec 侧**:多对象关系 / 字段映射 / 上下游消费 / 头部摘要 / 展开收起 / 卡片·表格·详情结构等命中时,复制 `templates/information-ownership-matrix.md`,运行 `scripts/check_information_ownership_matrix.py <gaps-or-spec.md>`。
- **spec-to-code 侧**:多对象 / 字段映射 / 卡片·表格·详情 / 头部·展开 UI 命中时,复制 `templates/product-structure-preflight.md`(额外含「当前页面不承载的信息」一项),运行 `scripts/check_product_structure_preflight.py <report-or-contract.md>`。
- 脚本 PASS = 结构/presence 合规,**不证明**归属/去重语义判对;业务判断交 judge/人。
- 命中以上结构但 SPEC 缺主对象 / 信息归属 / 头部职责 / 展开职责 / 去重规则时,**先回流 prd-to-spec**,不得用卡片、折叠、容器、边框、间距或样式去掩盖归属缺失。
