# P0 硬确认请求(Stop-on-P0 Confirmation Request)

> 用法:一旦出现 P0 硬确认项,停止生成完整 SPEC。只输出本确认请求、源对账摘要和占位骨架锚点;不得继续写 §0 具体契约值、单元详情、交互矩阵或验收断言。

## 停止原因

- 检测到 P0 硬确认项,继续写完整 SPEC 会把未确认产品判断包装成事实。
- 当前允许继续的工作:源对账、证据摘录、问题分组、推荐默认、占位骨架。
- 当前禁止继续的工作:冻结对象模型 / IA / 生命周期 / 权限 / id / 存储 / 消费读取 / 迁移策略,以及依赖这些值的单元详情和验收。

## 占位骨架

- `SPEC.md`: 仅允许保留章节标题和 `⚠️ 待确认 → P0-<n>` 占位。
- `notes/gaps.md`: 记录 P0/P1/P2 全量 gap。
- P0 解除后再回填 §0 契约、单元详情、交互矩阵和验收断言。
