# 完整性矩阵

> One table per object. Every required cell must be non-empty or explicit `N/A + reason`.
> 状态声明(loading/加载态、反馈态 toast/提示)由「空态/默认/加载」「错误/失败/反馈」两列负责——逐格回答"加载时显示什么、成功/失败给什么反馈",空格 = 状态声明缺失 = gap。

### 对象：<object>

| 操作 ＼ 关注点 | 权限 | 空态/默认/加载 | 错误/失败/反馈 | 状态/生命周期 | 数据一致性/引用完整性 | 消费/读取影响 |
| --- | --- | --- | --- | --- | --- | --- |
| 查 |  |  |  |  |  |  |
| 增 |  |  |  |  |  |  |
| 改 |  |  |  |  |  |  |
| 删 |  |  |  |  |  |  |

> 列语义:
> - **空态/默认/加载**:空态/默认值 + 该操作加载/进行中态(in-flight 显示什么、骨架屏/loading/禁用、何时结束)。
> - **错误/失败/反馈**:校验/越权/冲突/超时/回滚处理 + 成功/失败后的反馈态(toast/提示/inline 错误/二次确认结果)。
